1.安装MySQL

linux

win11

  • **-p 3306:3306**：它将容器内部的 3306 端口（MySQL 默认服务端口）映射到你电脑（宿主机）的 3306 端口。这样你才能通过本机的 localhost:3306 访问到这个运行在容器里的数据库。
  • **--name mysql**：容器命名。给这个启动的容器起一个名字叫 mysql。以后你在管理容器（比如查看日志、停止容器）时，可以直接用这个名字，而不用去记那一长串随机的容器 ID。
  • --restart=always：设置容器在退出后总是自动重启。这实现了开机自启。当你重启电脑或 Docker 服务后，这个 MySQL 容器会自动运行，无需手动干预。
  • **--privileged=true**赋予容器几乎与宿主机 root 用户相同的权限。在 Windows 上通常是为了解决挂载目录时的文件权限问题，确保容器内的 MySQL 进程有权限读写你电脑上的 D:\Projects\... 文件夹。
  • **-v D:\...\log:/var/log/mysql**将你电脑上的 D:\Projects\container\mysql\log 文件夹映射到容器内的 /var/log/mysql 目录。即使容器被删除或重建，数据库产生的日志文件依然安全地保存在你的电脑硬盘上，方便排查问题。
  • **-v D:\...\data:/var/lib/mysql**将你电脑上的 D:\Projects\container\mysql\data 文件夹映射到容器内的 /var/lib/mysql 目录。这是存放你所有数据库表数据的地方。如果没有这一行，一旦你删除了容器，你所有的数据库数据都会丢失。
  • **-v D:\...\conf:/etc/mysql/conf.d**将你电脑上的 conf 文件夹映射到容器内的 /etc/mysql/conf.d 目录。你可以把自定义的 .cnf 配置文件放在这个文件夹里，MySQL 启动时会自动读取它们，用来修改字符集、最大连接数等参数。
  • **-e MYSQL_ROOT_PASSWORD=123456**设置容器内的环境变量。这是初始化 MySQL 时必须设置的参数，用来指定数据库超级管理员（root）的密码为 123456。
  • **-d mysql:latest**表示让容器在后台运行（Detached 模式），不占用当前的命令行窗口。指定使用名为 mysql 的镜像，且版本标签为 latest（最新版）。

2.安装MongoDB

win11

  • -d: 让容器在后台运行。

  • -p 27017:27017: 将容器的 27017 端口映射到宿主机的 27017 端口，这是 MongoDB 的默认端口。

  • --name mongodb: 为容器指定一个名字，方便后续管理。

  • --restart=always: 设置容器重启策略为“总是”，确保 Docker 服务启动时，该容器也会自动启动，实现开机自启。

  • -e MONGO_INITDB_ROOT_USERNAME=admin: 设置 MongoDB 初始化管理员的账号为 admin。

  • -e MONGO_INITDB_ROOT_PASSWORD=123456: 设置管理员账号的密码为 123456。

  • -v ...

    : 将宿主机的文件夹挂载到容器内部，实现数据持久化。

    • D:\...\data:/data/db: 存放数据库文件。
    • D:\...\conf:/etc/mongo: 存放配置文件。
    • D:\...\log:/var/log/mongodb: 存放日志文件。

  • mongo:latest: 指定使用最新版本的 MongoDB 镜像。

  • --auth: 启用身份验证。设置此参数后，连接数据库时必须使用上面设置的账号和密码。
